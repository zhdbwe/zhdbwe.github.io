腾讯广告算法大赛2019

发表于 2019-04-20 | 更新于 2019-04-25 | 分类于 competion

数据处理阶段

ad_static_feature.out 广告静态数据

name type description
ad_id np.number 广告id,曝光日志中的广告id相关联[1,736053]
create_time np.number 创建时间,广告创建的时间戳,为0表示缺失[0,1.5e9]
count_id np.number 账户id,：广告 所在账户的唯一标识 ，账户结构分为四级：账户-推广计划-广告-素材[1,29743]
goods_id Object 商品id,广告推广目标的唯一标识,缺失值为-1
goods_type np.number 商品类型,广告推广目标的类型,枚举型
ad_type_id Object 广告类型id,广告所属的行业类别，缺失值为-1
material_size Object 素材大小,不同广告位对素材的尺寸要求不同，同一个广告可能有多个不同尺寸的素材,用逗号分割，缺失值为空，对nan填充之后缺失值为0

数据预处理，将goods_id 和 ad_type_id 转化为数字类型，如果转化失败直接删除该行。删除之前为735911，最删除之后数据量为727717。

ad_operation.dat 广告操作数据

name type description
ad_id np.number 广告id,曝光日志中的广告id相关联[1,736053]
create_time np.number 创建时间,广告创建的时间戳,为0表示缺失[0,2.0e13]
op_type np.number 操作类型，1-修改,2-创建
modify_field np.number 修改字段 :1-广告状态, 2-出价, 3-人群定向, 4-广告时段设置
modifyed_value Object 和之前的操作相关,广告状态0(失效),1(正常),出价(按分计算的整数),播出时段(7个64为uint_64转为二进制从低到高48位表示每半个小时是否投放),人群定向(字符串，特征名:特征值)

user_data 用户特征 (1296719)

name type description
user_id np.number 用户id,经过随机映射和上面的用户id对应
age np.number 年龄,经过随机映射
gender np.number 性别,男/女(1,2,3)可能还有第三种性别
area Object 省市的id,可以多值(逗号分割)
status Object 婚恋状态,可以多值(逗号分割)
education np.number 学历，范围(1,8)
con_ability np.number 消费能力(1,2,3)
device np.number 设备(0,4)
work Object 工作状态,多值
connect_type np.number 连接类型,无线/2G/3G/4G(1,5)
behavior Object 行为兴趣，多值

原始文件类型都为Object，将一些值转化为np.number。按10W条数据对文件分割，分为13个文件。

totalExposureLog.out 历史曝光数据

name type description
ad_request_id np.number 请求id，不唯一可能有多个请求id
time np.number 时间
ad_pos_id np.number 广告位id，脱敏之后
user_id np.number 用户id，和user_data中的user_id对应
ad_id np.number 广告id,区分不同广告，和广告特征文件中的广告id对应
ad_size np.number 广告的尺寸，同一个广告可能有不同尺寸
bid np.number 广告的出价,只记录cpc出价，非cpc出价，转化为cpc出价
pctr np.number 预估的pctr,basic_ecpm=pctr*bid
quality_ecpm np.number 将广告质量和用户体验转化为ecpm,主要影响因素有pctr/pvtr/窄定向等
total_ecpm np.number 广告排序的依据,total_ecpm=basic_ecpm+quality_ecpm
